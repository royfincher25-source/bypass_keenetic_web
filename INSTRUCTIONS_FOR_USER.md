# Инструкция по исправлению проблем обновления

## Проблемы, которые были исправлены:

1. **Кнопка "Обновить" не работает** - была ошибка 404 при попытке скачать файлы `bot3`
2. **Не отображается версия проекта** - файл `VERSION` не находился по правильному пути
3. **Ошибка `NameError: name 'jsonify' is not defined`** - отсутствовал импорт в `routes.py`

## Быстрое решение (рекомендуется):

Подключитесь к роутеру по SSH и выполните одну команду:

```bash
ssh root@192.168.1.1
cd /tmp
curl -sL -o fix.sh "https://raw.githubusercontent.com/royfincher25-source/bypass_keenetic_web/master/fix_all_problems_v2.sh"
chmod +x fix.sh
./fix.sh
```

## Что делает скрипт:

1. ✅ Исправляет имя репозитория: `bypass_keenetic-web` → `bypass_keenetic_web`
2. ✅ Исправляет ветку: `main` → `master`
3. ✅ Добавляет импорт `jsonify` в `routes.py`
4. ✅ Добавляет файл `VERSION` в `/opt/etc/web_ui/`
5. ✅ Удаляет ссылки на несуществующие файлы `bot3`
6. ✅ Устанавливает правильные права доступа
7. ✅ Перезапускает веб-интерфейс

## Проверка после исправления:

1. Откройте веб-интерфейс: `http://192.168.1.1:8080/service/updates`
2. Должна отображаться текущая версия проекта (например, "1.1.0")
3. Нажмите кнопку "Обновить" - она должна работать без ошибок

## Альтернативные варианты:

### Вариант 1: Обновить через веб-интерфейс
После выполнения скрипта выше, кнопка "Обновить" будет работать корректно и скачает последние файлы из репозитория.

### Вариант 2: Ручное обновление файлов
Если предыдущие варианты не сработают, следуйте инструкции в `docs/INSTRUCTION_MANUAL_UPDATE.md`.

## Если проблема сохраняется:

1. Проверьте логи веб-интерфейса: `tail -f /opt/var/log/web_ui.log`
2. Убедитесь, что у роутера есть доступ к интернету
3. Проверьте, что файлы были скачаны: `ls -la /opt/etc/web_ui/`
